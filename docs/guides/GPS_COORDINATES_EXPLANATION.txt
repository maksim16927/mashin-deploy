ОБЪЯСНЕНИЕ СИСТЕМЫ GPS КООРДИНАТ ДЛЯ ОБНАРУЖЕННЫХ ОБЪЕКТОВ

Система привязки GPS координат к обнаруженным объектам работает в три этапа.

ПЕРВЫЙ ЭТАП: ИЗВЛЕЧЕНИЕ НАЧАЛЬНЫХ КООРДИНАТ

Система в первую очередь пытается извлечь реальные GPS координаты из метаданных видеофайла. Для этого используется библиотека exifread, которая читает EXIF данные, встроенные в видео. Если в метаданных есть GPS теги GPSLatitude и GPSLongitude, система извлекает настоящие начальные координаты места съемки из самого видеофайла.

Только если GPS координаты не найдены в метаданных видео, система использует координаты по умолчанию как запасной вариант. В нашем случае это координаты Москвы 55.7558 градусов северной широты и 37.6173 градусов восточной долготы. Эти координаты используются исключительно когда система не может определить реальное местоположение из видео.

Важно понимать, что координаты Москвы это не основной источник данных, а лишь значения по умолчанию на случай отсутствия GPS информации в видео. Если видео содержит GPS метаданные, система всегда использует реальные координаты из видео, а не координаты Москвы.

ВТОРОЙ ЭТАП: ИНТЕРПОЛЯЦИЯ КООРДИНАТ ДЛЯ КАЖДОГО КАДРА

Для каждого кадра видео система вычисляет GPS координаты центра кадра. Это делается с помощью линейной интерполяции между начальными и конечными координатами.

Если известны начальные и конечные координаты маршрута, система использует простую формулу линейной интерполяции. Прогресс вычисляется как отношение номера текущего кадра к общему количеству кадров. Затем координаты вычисляются по формуле: начальная координата плюс разница между конечной и начальной координатой, умноженная на прогресс.

Если конечные координаты неизвестны, система вычисляет их на основе предполагаемой скорости движения. По умолчанию используется скорость 50 километров в час. Система вычисляет время видео в секундах, умножает на скорость и получает пройденное расстояние в метрах. Затем это расстояние конвертируется в градусы координат, учитывая что один градус широты примерно равен 111 километрам, а один градус долготы зависит от широты и равен 111 километрам умноженным на косинус широты.

ТРЕТИЙ ЭТАП: ВЫЧИСЛЕНИЕ КООРДИНАТ КОНКРЕТНОГО ОБЪЕКТА

Для каждого обнаруженного объекта система вычисляет его точные GPS координаты на основе его положения на кадре. Это самая сложная часть системы.

Система использует упрощенную модель камеры с известными параметрами. Фокусное расстояние камеры установлено как 700 пикселей, а высота камеры над землей как 5 метров. Эти параметры можно настроить для конкретной камеры для повышения точности.

Алгоритм работает следующим образом. Сначала вычисляется центр объекта на кадре как среднее арифметическое координат его ограничивающего прямоугольника. Затем вычисляется смещение этого центра от центра кадра в пикселях по горизонтали и вертикали.

Далее система преобразует это смещение в углы обзора камеры. Для этого используется функция арктангенса от отношения смещения к фокусному расстоянию. Это дает углы отклонения объекта от центра кадра.

Затем система вычисляет реальное смещение объекта от центра кадра в метрах на земле. Для этого используется высота камеры и тангенс вычисленных углов. Горизонтальное смещение вычисляется как высота умноженная на тангенс угла по горизонтали, вертикальное смещение как высота умноженная на тангенс угла по вертикали.

Наконец, эти смещения в метрах конвертируются в градусы координат. Вертикальное смещение конвертируется в изменение широты делением на 111000 метров на градус. Горизонтальное смещение конвертируется в изменение долготы делением на 111000 метров умноженных на косинус текущей широты, так как долгота зависит от широты.

Итоговые координаты объекта получаются добавлением этих изменений к координатам центра кадра.

ПРИМЕНЕНИЕ В СИСТЕМЕ

В процессе обработки видео для каждого обнаруженного объекта дорожный знак, барьер, элемент инфраструктуры система вызывает метод get_object_coordinates, передавая координаты ограничивающего прямоугольника объекта и номер кадра. Метод возвращает GPS координаты объекта, которые затем сохраняются в Excel отчет вместе с другой информацией об объекте.

ТОЧНОСТЬ СИСТЕМЫ

Точность вычисления координат зависит от нескольких факторов. Во первых, от точности начальных координат. Если они извлечены из метаданных видео, точность высокая. Если используются координаты по умолчанию, точность снижается.

Во вторых, точность зависит от правильности параметров камеры. Фокусное расстояние и высота камеры должны соответствовать реальным значениям для конкретной камеры. В текущей реализации используются усредненные значения.

В третьих, система использует упрощенную модель камеры, которая предполагает плоскую землю и отсутствие искажений объектива. Для повышения точности можно использовать более сложные модели с учетом дисторсии объектива и рельефа местности.

В четвертых, интерполяция координат между кадрами предполагает равномерное движение по прямой линии. В реальности движение может быть неравномерным и по кривой траектории, что снижает точность интерполяции.

Несмотря на эти ограничения, система обеспечивает достаточную точность для практических задач, таких как создание карты расположения дорожных объектов или навигация к обнаруженным объектам. Координаты объектов позволяют точно определить их местоположение на карте и использовать эту информацию для различных целей.
