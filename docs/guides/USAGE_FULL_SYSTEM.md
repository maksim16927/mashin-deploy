# 🚀 Использование полной системы детекции

## Обзор

Комплексная система детекции использует все доступные модели одновременно для обработки видео:

1. **Дорожные знаки (RTSD-155)** - модель YOLOv8s с 155 классами российских знаков
2. **Инфраструктура (COCO)** - модель YOLOv8n для детекции:
   - Автобусы (автобусные остановки)
   - Машины, грузовики
   - Светофоры
   - Знаки остановки
   - Люди, велосипеды
3. **П-образные опоры** - эвристический метод детекции опор над дорогой
4. **Дорожная разметка** - OpenCV (Canny + HoughLinesP)

## Быстрый старт

### 1. Загрузка моделей

Загрузите все необходимые модели из интернета:

```bash
python scripts/download_all_models.py
```

Этот скрипт загрузит:
- YOLOv8n (COCO) - базовая модель для инфраструктуры
- YOLOv8s (COCO) - более точная модель
- YOLOv8n-seg - сегментационная модель

**Примечание:** Модель дорожных знаков `yolov8s_35epochs_rtsd155.pt` уже должна быть в папке `models/`

### 2. Обработка одного видео

```bash
python scripts/detect_full_system.py "videos/original/video.mp4"
```

С параметрами:

```bash
python scripts/detect_full_system.py "videos/original/video.mp4" \
    -o "videos/detected/result.mp4" \
    --conf 0.25
```

### 3. Обработка всех оригинальных видео

Обработать все видео из `videos/original/` автоматически:

```bash
python scripts/process_all_videos_full.py
```

Этот скрипт:
- Найдет все видео в `videos/original/`
- Обработает каждое со всеми моделями
- Сохранит результаты в `videos/detected/`

## Детекция объектов

### Дорожные знаки (RTSD-155)

- **Модель:** `models/yolov8s_35epochs_rtsd155.pt`
- **Классы:** 155 российских дорожных знаков
- **Цвет детекции:** 🟢 Зеленый

### Инфраструктура (COCO)

Детектирует:
- 🟡 **Автобусы** - могут указывать на остановки
- 🔵 **Машины** - транспорт
- 🔴 **Знаки остановки** - stop sign
- 🔵 **Светофоры** - traffic light
- 🟠 **Грузовики** - truck
- 🟣 **Мотоциклы** - motorcycle
- 🟣 **Велосипеды** - bicycle
- ⚪ **Люди** - person

### П-образные опоры

- **Метод:** Эвристический поиск горизонтальных структур
- **Область:** Верхняя треть кадра
- **Цвет детекции:** 🟣 Фиолетовый

**Примечание:** Детекция опор использует простую эвристику. Для более точной детекции рекомендуется использовать специализированные модели или дообучить на своих данных.

### Дорожная разметка

- **Метод:** OpenCV (Canny + HoughLinesP)
- **Область:** Нижняя половина кадра (дорога)
- **Цвет детекции:** 🔵 Синий

## Параметры

### Порог уверенности (`--conf`)

- **По умолчанию:** 0.25
- **Диапазон:** 0.0 - 1.0
- **Рекомендации:**
  - `0.25` - баланс между точностью и полнотой
  - `0.3-0.4` - выше точность, меньше ложных срабатываний
  - `0.2-0.25` - выше полнота, больше детекций

### Выходной файл (`-o, --output`)

- **По умолчанию:** `videos/detected/{имя_видео}_FULL_DETECTION.mp4`
- **Формат:** MP4

## Советы

1. **Для лучшей детекции П-образных опор:**
   - Используйте видео высокого разрешения
   - Опоры должны быть четко видны в верхней части кадра

2. **Для ускорения обработки:**
   - Используйте более легкие модели (yolov8n вместо yolov8s)
   - Увеличьте порог уверенности для фильтрации слабых детекций

3. **Для большей точности:**
   - Используйте yolov8s вместо yolov8n
   - Уменьшите порог уверенности до 0.2-0.25

## Устранение проблем

### Модель не найдена

```bash
# Проверьте наличие модели
ls models/yolov8s_35epochs_rtsd155.pt

# Загрузите модели
python scripts/download_all_models.py
```

### Видео не открывается

- Проверьте путь к видео
- Убедитесь, что формат видео поддерживается (MP4, MOV, AVI)
- Проверьте кодек видео

### Низкая скорость обработки

- Используйте GPU (CUDA/MPS) вместо CPU
- Уменьшите разрешение видео
- Обрабатывайте только каждый N-ый кадр
